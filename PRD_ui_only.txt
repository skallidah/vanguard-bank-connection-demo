PRODUCT REQUIREMENTS DOCUMENT
Vanguard Bank Connection Demo — KaneAI UI Test Generation
================================================================================

DOCUMENT PURPOSE
----------------
This PRD defines the UI workflow, pages, and acceptance criteria for the
Vanguard Bank Connection Demo app. It is written exclusively to drive UI
test scenario and test case generation in LambdaTest KaneAI — covering
positive (happy path), negative (validation/error), and edge case scenarios.
All tests interact with the browser only; no direct API calls are made by tests.

LINKS
-----
  App URL:  https://vanguard-bank-connection-demo-production.up.railway.app
  GitHub:   https://github.com/skallidah/vanguard-bank-connection-demo

--------------------------------------------------------------------------------
SECTION 1: APPLICATION OVERVIEW
--------------------------------------------------------------------------------

The Vanguard Bank Connection Demo simulates the bank account linking flow from
vanguard.com. A customer connects an external bank account to their Vanguard
investment accounts through a 4-step guided UI. At the end, the system
initiates micro-deposit verification.

User journey (5 pages):
  Landing → Add Bank → Authorize Accounts → Review → Success

The database resets to seed data on each Railway deployment, so the five
pre-seeded draft records (draft_001 through draft_005) are always available.

SECTION 3: PAGES & UI ELEMENTS
--------------------------------------------------------------------------------

PAGE 1 — Landing
  URL:      /
  Title:    "Connect your bank account"
  Elements:
    - "Vanguard" logo text at the top
    - Heading: "Connect your bank account"
    - Subheading paragraph starting with "Quickly and easily move money..."
    - Red-text paragraph starting with "Follow these steps..."
    - Ordered list with 4 steps describing the micro-deposit process
    - Bold important note paragraph about timing restrictions
    - Button: "CONTINUE" (navigates to /add-bank)
    - Footer: "Vanguard Internal Use Only"

PAGE 2 — Add Bank
  URL:      /add-bank
  Title:    "Add a new bank"
  Elements:
    - Heading: "Add a new bank"
    - Subheading: "Bank information"
    - Routing number input (required, must be exactly 9 digits)
    - Bank name display (auto-populated on valid routing number entry, read-only)
    - Account number input (required)
    - Re-enter account number input (required, must match account number)
    - Correspondent routing number input (read-only, auto-populated if applicable)
    - Correspondent account number input (read-only, auto-populated if applicable)
    - Account type radio buttons: "Checking" (default selected) | "Savings"
    - Owner section labeled "Personal account owner"
      - First name input (required)
      - Middle name input (optional)
      - Last name input (required)
      - Suffix dropdown (optional): --, Mr., Mrs., Ms., Miss, Dr., Prof.,
        Jr., Sr., II, III
    - "+ Add another name" link (optional, adds additional owner)
    - Bank nickname input (optional)
    - "Make this your primary bank" checkbox (checked by default)
    - Button: "CANCEL" (navigates to /)
    - Button: "CONTINUE" (validates and submits; navigates to /authorize/<id>)
    - Footer: "Vanguard Internal Use Only"
    - "API Calls" log panel at the bottom of the page
  Validation errors (shown inline without page reload):
    - Routing number blank → "Routing number is required."
    - Routing number not 9 digits or contains letters → "Must be exactly 9 digits."
    - Account number blank → "Account number is required."
    - Re-enter account number blank → "Please re-enter account number."
    - Re-enter account number differs → "Account numbers do not match."
    - Owner first or last name blank → "Owner first and last name are required."

PAGE 3 — Authorize Accounts
  URL:      /authorize/<draft_id>
  Title:    "Authorize accounts"
  Elements:
    - Heading: "Authorize accounts"
    - Descriptive paragraph about choosing Vanguard accounts
    - Subheading: "Bank information"
    - Data table with rows: Bank name, Routing number, Account number,
      Account type, Account owners (and optionally Correspondent routing
      number, Correspondent account number if present for the bank)
    - Subheading: "Accounts that will be authorized to use this bank account"
    - Descriptive paragraph about identical ownership
    - List of auto-authorized accounts in format:
        <owner name>—<account name>—<VA ID> (Managed)   [if managed]
        <owner name>—<account name>—<VA ID>              [if not managed]
      or: "No auto-authorized accounts." if none
    - Subheading: "Review additional accounts that can be authorized to use this bank"
    - Descriptive paragraph about non-identical ownership
    - List of eligible additional accounts (same format as above)
      or: "No eligible additional accounts." if none
    - Button: "CANCEL" (navigates to /)
    - Button: "BACK" (navigates to /add-bank)
    - Button: "REVIEW" (saves authorizations and navigates to /review/<id>)
    - "API Calls" log panel at the bottom of the page

PAGE 4 — Review
  URL:      /review/<draft_id>
  Title:    "Review your information"
  Elements:
    - Heading: "Review your information"
    - Intro paragraph: "The bank below will be connected to your account..."
    - Bold warning: "Important note: You can't move money from Vanguard..."
    - Info box with 3-step next-steps instructions
    - Subheading: "Part 1 details"
    - "Bank information" bordered section with Edit link and data table:
        Bank name, Routing number, Account type, Account owners,
        Bank nickname (only if nickname was set), Primary bank (Yes/No),
        Verification method (always "MICRO_DEPOSIT")
    - "Accounts authorized to use this bank" bordered section:
        List of authorized accounts in format:
          <owner name>—<account name>—<VA ID> (Managed)
        or: "No authorized accounts." if none
    - Button: "CANCEL" (navigates to /)
    - Button: "BACK" (navigates to /authorize/<draft_id>)
    - Button: "SUBMIT" (submits and navigates to /success/<bc_id>)

PAGE 5 — Success
  URL:      /success/<bc_id>
  Title:    "Submission Successful"
  Elements:
    - "Vanguard" logo text
    - Large green checkmark (✓)
    - Heading: "Entry submitted successfully"
    - Paragraph: "Your bank account has been connected and micro-deposit
      verification has been initiated."
    - Paragraph: "It may take 1–3 business days for the deposits to appear
      in your account."
    - Bank Connection ID in monospace text: "Bank Connection ID: bc_xxxxxx"
    - Button: "START OVER" (navigates to /)
    - Footer: "Vanguard Internal Use Only"
